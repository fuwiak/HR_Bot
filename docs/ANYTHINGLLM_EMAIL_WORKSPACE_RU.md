# AnythingLLM: новая рабочая область для проверки и ответов на письма Yandex

Краткая инструкция: как создать в AnythingLLM **новую рабочую область** («Новая Рабочая Область») и использовать её для **проверки почты Yandex** и **ответов на письма**.

---

## ⚠️ Важно: где видеть письма и как на них отвечать

**Рабочая область AnythingLLM (email-workspace) не подключается к почте.** К Yandex подключается только **Telegram-бот** (по IMAP). Поэтому:

| Где вы находитесь | Что происходит |
|-------------------|-----------------|
| **Чат AnythingLLM со скиллом Email Manager** | Можно **смотреть письма** и **отвечать**: напишите **mail письма**, **письмо 1**, **mail ответь на письмо 2: текст**. Скилл обращается к backend HR_Bot за списком писем и отправкой ответа. |
| **Чат AnythingLLM без скилла / без вызова скилла** | Модель сама **не видит** почту. Если написать «покажи последний mail» без вызова скилла, ответит вроде «не вижу писем». Нужно включить скилл «Email Manager» и писать команды (mail письма, письмо 1 и т.д.). |
| **Telegram (бот / канал лидов)** | Здесь тоже **видно** письма: уведомления приходят от бота (по `/email_check` или автоматически), кнопка **«📝 Подготовить ответ»**, отправка ответа через бота. |

**Итого:** смотреть письма и отвечать можно и в **AnythingLLM** (скилл Email Manager: команды mail письма, письмо 1, ответь на письмо N), и в **Telegram** (команда `/email_check`, уведомления в канал, кнопка «Подготовить ответ»).

## Смотреть письма и отвечать прямо в AnythingLLM (скилл Email Manager)

Добавлен скилл **Email Manager**: в чате AnythingLLM можно **просматривать письма** и **отправлять ответы**, не переходя в Telegram.

### Что нужно

1. **Backend HR_Bot** доступен по URL и у него настроена почта Yandex (`YANDEX_EMAIL`, `YANDEX_IMAP_PASSWORD` и т.д.).
2. В AnythingLLM в настройках скилла **«Email Manager»** (Setup) указан **HR_BOT_BACKEND_URL** = URL бэкенда **без слэша в конце**. Либо переменная окружения AnythingLLM.
3. В настройках агента включён скилл **«Email Manager»**.

4. **Чтобы агент не «галлюцинировал» письма** (не придумывал Иван Петров, Анна Смирнова и т.д.), в **System Prompt** агента (Workspace → Agent → System Prompt) добавьте:
   ```
   Когда пользователь просит показать письма, почту, mail, письма, письмо N, ответить на письмо — ты ОБЯЗАН вызвать скилл "Email Manager" с параметрами (action: list или get или reply, limit, email_number, reply_text) и вернуть пользователю ТОЛЬКО результат скилла. Запрещено придумывать письма, имена, темы или даты. Если скилл вернул пустой список — скажи, что писем нет или произошла ошибка.
   ```

Скилл использует формат AnythingLLM **entrypoint** с параметрами: агент вызывает его с `action` (list / get / reply), `limit`, `email_number`, `reply_text`. Подробнее: **`plugins/agent-skills/email-manager/README.md`**.

### Что писать в чате

Агент сам решает, когда вызвать скилл. Достаточно написать, например:

- **«Покажи последние 5 писем»** / **«Проверь почту»** — список писем;
- **«Покажи письмо номер 1»** — полный текст последнего письма;
- **«Ответь на письмо 2: Добрый день, высылаю информацию.»** — отправить ответ.

Параметры вызова (action, limit, email_number, reply_text) агент подставляет по примерам из скилла. Таким образом, в AnythingLLM можно **смотреть письма** и **отвечать** на них без Telegram.

## 1. Проверка писем Yandex (уже работает)

Проверка почты в боте уже настроена:

- **Фоновая проверка** — по интервалу `EMAIL_CHECK_INTERVAL` (у вас 10 секунд; для продакшена часто ставят 300).
- **Команда** — `/email_check`: проверить новое письмо и отправить уведомление в Telegram.
- **Переменные** — `YANDEX_EMAIL`, `YANDEX_IMAP_PASSWORD`, `YANDEX_IMAP_SERVER`, `YANDEX_IMAP_PORT` (у вас уже заданы).

Новые письма приходят в канал лидов с кнопками: **«📝 Подготовить ответ»**, **«📄 С КП»** и т.д. Дополнительно настраивать AnythingLLM для *проверки* писем не нужно — это делает бот через IMAP.

## 2. Новая рабочая область в AnythingLLM для ответов на письма

Чтобы **ответы на письма** (черновики по кнопке «Подготовить ответ» и команде `/email_draft`) строились на базе **отдельной** рабочей области AnythingLLM:

### Шаг 1. Создать workspace в AnythingLLM

1. Откройте веб-интерфейс AnythingLLM: **https://anastassiya-hr-bot.up.railway.app** (или ваш URL).
2. Войдите в систему.
3. **Workspaces** → **Create new workspace**.
4. Укажите имя, например: **Новая Рабочая Область** (или «Ответы на письма»).
5. Задайте **slug** латиницей, например: `email-workspace` или `email-support`.  
   Slug понадобится для переменной окружения.
6. Сохраните workspace.

### Шаг 2. Настроить workspace (по желанию)

- **Документы** — загрузите в эту область документы, на которые должен опираться бот при ответах (FAQ, регламенты, шаблоны ответов и т.д.).
- **LLM / Embedder** — обычно наследуются из общих настроек AnythingLLM (Settings).
- **System Prompt** — можно задать, чтобы модель не путалась, когда бот присылает письмо:  
  *«Ты помощник по ответам на входящие письма. Тебе приходит тема и текст письма — сформируй краткий вежливый черновик ответа (без приветствия в начале, только суть). Не проси прислать письмо — ты получаешь его в том же сообщении.»*  
  (В чате AnythingLLM вручную письма не передаются — только бот в Telegram шлёт контекст при нажатии «Подготовить ответ».)

### Шаг 3. Переменные окружения HR_Bot (Telegram/Backend)

В Railway (или в `.env`) для сервиса, где запущен бот/backend, должны быть заданы:

```env
USE_ANYTHINGLLM_RAG=true
ANYTHINGLLM_BASE_URL=https://anastassiya-hr-bot.up.railway.app
ANYTHINGLLM_API_KEY=your_anythingllm_api_key
ANYTHINGLLM_WORKSPACE_SLUG=hr-bot
```

Для ответов именно из **новой** рабочей области добавьте:

```env
ANYTHINGLLM_EMAIL_WORKSPACE_SLUG=email-workspace
```

Замените `email-workspace` на тот **slug**, который вы указали при создании workspace в п. 1.

- Если `ANYTHINGLLM_EMAIL_WORKSPACE_SLUG` **не задан** — для черновиков ответов на письма используется основной workspace (`ANYTHINGLLM_WORKSPACE_SLUG`, например `hr-bot`).
- Если **задан** — черновики ответов на письма генерируются через указанный email-workspace.

### Шаг 4. Где это используется в боте

- Кнопка **«📝 Подготовить ответ»** в уведомлении о письме — черновик запрашивается у AnythingLLM (ваша новая рабочая область, если задан `ANYTHINGLLM_EMAIL_WORKSPACE_SLUG`).
- Команда **`/email_draft [текст запроса]`** — то же самое.

Если AnythingLLM недоступен или вернул пустой ответ, бот подставляет черновик через прежнюю логику (`generate_proposal`).

## 3. Краткая схема

| Действие | Кто выполняет |
|----------|----------------|
| Проверка почты Yandex | Бот (IMAP, `EMAIL_CHECK_INTERVAL`, `/email_check`) |
| Уведомления о новых письмах | Бот → канал лидов в Telegram |
| **Смотреть письма / отвечать в AnythingLLM** | Скилл **Email Manager** (команды: mail письма, письмо 1, mail ответь на письмо N: текст) |
| Черновик ответа («Подготовить ответ» в Telegram) | AnythingLLM workspace из `ANYTHINGLLM_EMAIL_WORKSPACE_SLUG` (или основной workspace) |
| Отправка ответа на email | В Telegram: пользователь вводит текст → бот отправляет. В AnythingLLM: скилл Email Manager → backend `/api/email/reply` |

---

## 4. Проверка

1. В AnythingLLM создана новая рабочая область с нужным slug.
2. В Variables бота задан `ANYTHINGLLM_EMAIL_WORKSPACE_SLUG=<ваш-slug>`.
3. Вызвать `/email_check` или дождаться фоновой проверки, нажать «📝 Подготовить ответ» — черновик должен приходить из новой рабочей области (при включённом AnythingLLM и корректном API ключе).

API-ключ AnythingLLM берётся из **Settings → Developer API** в веб-интерфейсе AnythingLLM; тот же ключ задаётся в `ANYTHINGLLM_API_KEY`.
