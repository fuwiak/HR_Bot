# Сохранение данных AnythingLLM на Railway (обязательно)

**Проблема:** после каждого коммита и деплоя сбрасываются workspace, пароль и анкета AnythingLLM.

**Причина:** при деплое Railway поднимает новый контейнер. Без **постоянного тома (Volume)** каталог `/app/server/storage` каждый раз пустой — все данные (SQLite, документы, настройки) теряются.

**Решение:** один раз добавить Volume для сервиса Frontend (AnythingLLM) в Railway.

---

## Шаги (сделать один раз)

1. Откройте [Railway Dashboard](https://railway.app/dashboard) → ваш проект.
2. Выберите сервис **Frontend** (AnythingLLM).
3. Перейдите в **Variables** или откройте панель сервиса.
4. Найдите раздел **Volumes** (или через Command Palette: `⌘K` / `Ctrl+K` → «Add Volume»).
5. **Добавьте том:**
   - **Mount Path:** `/app/server/storage` (обязательно именно так).
   - Имя тома можно любое, например `anythingllm-storage`.
6. Сохраните. Сделайте **Redeploy** сервиса Frontend.

После этого при каждом новом деплое Railway будет подключать тот же том к новому контейнеру — пароль, workspace, документы и анкета сохранятся.

---

## Проверка

После добавления тома в настройках сервиса Frontend в разделе Volumes должен быть указан путь `/app/server/storage`. При следующих деплоях данные AnythingLLM не будут теряться.

---

## Если том уже добавлен, но данные всё равно сбрасываются

- Убедитесь, что **Mount Path** именно `/app/server/storage` (без опечаток).
- Убедитесь, что том привязан к сервису **Frontend** (AnythingLLM), а не к другому сервису.
- В логах деплоя не должно быть ошибок монтирования; при необходимости сделайте Redeploy.
